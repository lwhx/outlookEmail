# OutlookEmail 浏览器扩展使用说明

这是 OutlookEmail 的 Chrome / Edge 浏览器扩展。扩展使用 Web 端登录密码建立正常 Web Session，不使用对外 API Key；所有管理请求仍走服务端现有的 Session + CSRF 保护。

扩展默认在浏览器侧边栏中运行，不会切换当前网页标签页。

## 适用浏览器

- Google Chrome，建议使用较新的稳定版。
- Microsoft Edge，建议使用较新的稳定版。
- 需要浏览器支持 Manifest V3 和 Side Panel。

## 安装扩展

1. 打开浏览器扩展管理页：
   - Chrome: `chrome://extensions/`
   - Edge: `edge://extensions/`
2. 打开“开发者模式”。
3. 点击“加载已解压的扩展程序”。
4. 选择本仓库里的 `browser-extension` 目录。
5. 安装完成后，工具栏会出现 `OutlookEmail` 扩展入口。

## 更新扩展

代码更新后，需要在扩展管理页手动重新加载：

1. 打开 `chrome://extensions/` 或 `edge://extensions/`。
2. 找到 `OutlookEmail 控制台`。
3. 点击“重新加载”按钮。
4. 重新打开侧边栏。

如果侧边栏仍显示旧界面，可以关闭侧边栏后重新点击扩展图标。

## 首次配置

1. 点击浏览器工具栏里的 OutlookEmail 扩展图标，打开侧边栏。
2. 填写“服务地址”，例如：
   - 本地运行：`http://127.0.0.1:5000`
   - 服务器部署：`https://your-domain.example`
3. 填写“Web 登录密码”，也就是登录 OutlookEmail Web 页面使用的密码。
4. 按需勾选“记住密码到本机浏览器”。
5. 点击“保存配置”，或直接点击上方功能入口开始使用。

说明：

- 如果勾选“记住密码到本机浏览器”，密码会保存在当前浏览器本地扩展存储中。
- 如果不勾选，密码只放在浏览器会话级存储中；浏览器会话结束后需要重新输入。
- 扩展第一次访问服务地址时，浏览器可能弹出站点访问权限确认，需要允许。

## 日常使用

打开侧边栏后，可以直接点击顶部功能入口：

- `邮箱`
- `导入`
- `刷新`
- `Token`
- `导出`
- `标签`
- `设置`

侧边栏右上角的齿轮按钮可以展开或收起配置区。`刷新`按钮会重新加载当前功能页。

## 邮箱

邮箱页用于查看和管理普通邮箱、临时邮箱以及 Cloudflare 全部邮件。

主要能力：

- 选择邮箱分组，扩展会记住上次选择的分组。
- 搜索普通邮箱，支持邮箱、备注、标签或别名。
- 普通邮箱列表支持分页。
- 点击账号行的“邮件”直接查看邮件列表。
- 邮件列表显示在账号列表上方，不需要在大量账号里向下滑很久。
- 邮件详情支持查看正文、附件入口、标记已读、删除普通邮件。
- 普通账号的其他操作收在“更多操作”里：编辑、标签、开启/关闭转发、刷新 Token、复制、删除。

临时邮箱：

- 进入“临时邮箱”分组后，会加载 `/api/temp-emails` 的临时邮箱列表。
- 支持 GPTMail、DuckMail、Cloudflare 临时邮箱。
- 可查看单个临时邮箱邮件。
- 可刷新、复制、删除临时邮箱。
- 如果配置了 Cloudflare 临时邮箱服务，会显示 `Cloudflare所有邮件` 入口，可查看当前 Worker 中的全部邮件，并按收件地址过滤。

## 导入

导入页支持普通邮箱和临时邮箱。

普通邮箱：

- 支持 Outlook / Hotmail。
- 支持 Gmail、QQ、163、126、Yahoo、阿里邮箱等 IMAP 邮箱。
- 支持自定义 IMAP。
- 可选择目标分组、导入格式、统一备注、状态、是否开启转发和导入时绑定标签。

常见格式：

```text
邮箱----密码----ClientID----RefreshToken
```

自定义 IMAP 格式：

```text
邮箱----IMAP密码----IMAP主机----IMAP端口
```

临时邮箱：

- `gptmail`: 每行一个邮箱。
- `duckmail`: 每行 `邮箱----密码`。
- `cloudflare`: 每行 `邮箱----JWT`。

生成临时邮箱：

- 支持 GPTMail / DuckMail / Cloudflare。
- DuckMail 需要填写域名、用户名和密码。
- Cloudflare 可以填写域名和用户名；用户名可留空随机生成。

## 刷新

刷新页用于管理 Outlook 账号的 Refresh Token 刷新。

主要能力：

- 查看刷新统计。
- 按状态筛选：全部、成功、失败、从未刷新。
- 搜索待刷新账号。
- 单账号刷新。
- 单账号重试。
- 全量刷新。
- 流式重试失败账号。
- 快速重试失败账号。
- 停止全量刷新。
- 查看刷新日志和失败列表。

全量刷新和流式重试会通过 SSE 显示进度。

## Token

Token 页用于辅助获取 Microsoft OAuth Refresh Token，并可直接保存为账号。

使用流程：

1. 复制授权链接。
2. 在浏览器中完成 Microsoft 授权。
3. 复制授权后的完整回调 URL。
4. 回到侧边栏，填写邮箱、账号密码、目标分组。
5. 粘贴完整回调 URL。
6. 点击“换取并预览”。
7. 确认无误后点击“保存账号”。

保存账号时会按 Outlook 导入格式调用服务端账号导入接口。

## 导出

导出页支持二次验证后导出账号。

使用流程：

1. 输入 Web 登录密码作为二次验证密码。
2. 选择导出全部，或勾选分组后导出选中分组。
3. 导出内容会显示在文本框中。
4. 可点击“复制结果”复制导出内容。

导出临时邮箱分组时，会按服务端导出逻辑包含对应临时邮箱内容。

## 标签

标签页支持：

- 创建标签。
- 删除标签。
- 给普通账号批量添加或移除标签。
- 给临时邮箱批量添加或移除标签。

批量绑定时需要填写账号 ID 或临时邮箱 ID。ID 可以在邮箱列表中看到。

## 设置

设置页覆盖常用 Web 设置：

- 基础刷新设置。
- Cron 和应用时区。
- 临时邮箱服务配置。
- DuckMail 配置。
- Cloudflare Worker 域名、邮箱域名、管理密码。
- 转发渠道。
- SMTP / Telegram / 企业微信配置。
- WebDAV 备份配置。
- 对外 API Key 查看和修改。

设置页中的敏感配置会直接写入服务端设置接口。修改 WebDAV 备份等敏感设置时，需要填写对应验证密码。

## 登录机制

扩展登录流程如下：

1. 扩展调用 `POST /api/extension/login`。
2. 服务端验证 Web 登录密码。
3. 服务端返回 60 秒有效的一次性 `launch_url`。
4. 扩展访问 `launch_url`，服务端在自身域名下写入正常 Web Session。
5. 后续扩展请求使用该 Session，并通过 `GET /api/csrf-token` 获取 CSRF Token。

如果服务端版本太旧，没有 `/api/extension/login`，扩展会尝试回退到 `/login` 密码登录接口。

## 常见问题

### 点击功能后一直显示正在登录

检查：

- 服务地址是否正确，不能多写路径，例如应填写 `https://your-domain.example`，不要填写 `https://your-domain.example/#settings`。
- Web 登录密码是否正确。
- 浏览器是否允许扩展访问该服务地址。
- 服务端是否可以正常打开。
- 部署反向代理是否正确转发 Cookie。

### 提示需要允许访问服务地址

这是浏览器扩展的站点权限提示。允许后扩展才能请求你的 OutlookEmail 服务。

### 临时邮箱分组显示暂无账号

临时邮箱分组不读取普通账号表，而是读取 `/api/temp-emails`。如果 Web 端有临时邮箱但扩展没有显示，先在扩展里点击“刷新”，再确认当前服务地址连接的是同一个服务端实例。

### Cloudflare所有邮件没有内容

检查设置页中的 Cloudflare 配置：

- Cloudflare Worker 域名。
- Cloudflare 邮箱域名。
- Cloudflare 管理密码。

配置后，在邮箱页进入“临时邮箱”分组，点击 `Cloudflare所有邮件`。

### 扩展更新后界面没变化

需要在浏览器扩展管理页点击“重新加载”。只刷新网页不会更新已加载的扩展代码。

### 不想保存密码

取消勾选“记住密码到本机浏览器”。这样密码只保存在浏览器会话级存储中，关闭浏览器会话后需要重新输入。

## 文件说明

- `manifest.json`: Chrome Manifest V3 清单。
- `background.js`: 点击扩展图标时打开侧边栏。
- `sidepanel.html` / `sidepanel.css` / `sidepanel.js`: 侧边栏控制台。
- `popup.html` / `popup.css` / `popup.js`: 兼容弹窗入口。
- `api-client.js`: 登录、Session、CSRF、JSON/Text/SSE 请求封装。
- `storage.js`: 本地配置和界面状态存储。
